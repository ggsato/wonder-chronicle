# Wonder Chronicle MVP Implementation Decision Log

## Spec Delta

- 変更点:
  実装基盤を `React + Vite + TypeScript` に固定した。詳細画面は PC / tablet では右側の固定パネルを基本にし、mobileでは下から開く詳細シートへ切り替えた。mobile上部は `Selected Period` と `追加する` / `編集する` を優先し、粒度切替と表示オプションはシートへ分離した。年表上の3Wラベルは日本語の縦見出しへ切り替え、期間表示は `Selected Period` のみを採用した。
- 理由:
  ワークスペースに既存アプリがなかったため、最小構成で縦書きUIと横スクロール検証を早く成立させる必要があった。加えて、Issue #1 で mobileは「閲覧操作の集約点」ではなく「最短で記録に入る入口」として再設計する方針が固まった。
- 実装への影響:
  ルーティングを増やさず `TimelinePage` 内で状態遷移を閉じた。粒度ごとの選択状態を保持しつつ、明示選択時には他粒度へ同期する構成にした。mobileでは `Selected Period` タップで粒度シート、`表示` ボタンで表示オプションシートを開く構成にした。詳細画面は period 単位の最小版として実装し、PC / tablet では固定パネル、mobileでは詳細シートとして再利用した。ドラッグスクロールと列クリックを両立させた。
- 将来戻すか/このまま採用するか:
  MVPではこのまま採用。Phase 2で別ページ化や、week / month 詳細の専用閲覧導線を再評価する。

## Visual Rules Adopted

- 列間は `12px` を採用した。
- 画像最大高は `140px` に調整した。
- 日付ラベルは横組みタグで表示した。
- 3Wラベルは `願い / 感動 / 問い` の日本語見出しに変え、文字を縦に積んで表示した。
- 写真は列上段の固定スロットに置き、写真なしでも高さを揃えるプレースホルダを出した。
- PC / tabletでは詳細を開いていない状態でも右ペインを残し、年表が主役の画面構成を維持した。
- mobileでは上部の操作密度を下げ、CTA と期間表示を先に見せる構成を採用した。

## Verification Notes

- `npm run build` で型チェックと本番ビルドが通過した。
- day / week / month の列数変化をサンプルデータで確認した。
- direction 切替時は `past-right` のときだけ描画順を反転し、selected period の中央寄せで意味保持を確認した。
- 粒度切替時は `Selected Period` を基準に、対応する週/月/日へ同期しつつ、切替後も対象 period が画面内に見えることを確認した。
- 日付変更時に保存先へ既存記録がある場合は保存エラーにする構成へ変更した。
- ドラッグスクロールと列クリックが両立する構成にした。
